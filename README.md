# Construction Pendency Tracker

A high-performance internal web application built for real-estate construction teams to track, resolve, and audit open action items ("pendencies"), approvals, and decision dates across project towers.

Replaces error-prone Excel pendency trackers with automatic audit trails for every missed/pushed target date (**CBE date** — Committed By Estimate date), computed delivery states, and real-time live sync.

---

## Technical Stack

- **Frontend**: Next.js 15 (App Router, TypeScript) compiled via static HTML export (`output: 'export'`)
- **Styling**: Tailwind CSS v4 with warm off-white canvas theme tokens
- **Database & Realtime Sync**: Supabase (Postgres with automatic Postgres triggers, view calculation, and Supabase Realtime channels)
- **Deployment**: Cloudflare Pages (hosted as static assets on Cloudflare's Edge CDN)
- **Data Table**: TanStack Table v8 + `@tanstack/react-virtual` for virtualized fast editing
- **Data Charts**: Recharts
- **Export Engine**: SheetJS (`xlsx`)
- **Notifications**: Client-side Browser Notification API + Service Worker (no push server required)

---

## Core Data Model & Derived Fields

Calculations are computed structurally at the database view level (`v_pendency_dashboard`) to eliminate data drift:

| Computed Field | Derivation Logic |
|---|---|
| `days_open` | `CURRENT_DATE - opened_on` |
| `is_overdue` | `status = 'open' AND current_cbe_date < CURRENT_DATE` |
| `days_since_cbe_due` | `CURRENT_DATE - current_cbe_date` (when overdue) |
| `on_track_status` | `Open + No CBE` → **Awaiting CBE**<br>`Open + CBE Future` → **On Track**<br>`Open + CBE Past` → **Delayed**<br>`Status Closed` → **Closed** |
| `cbe_change_count` | Calculated via `cbe_history` table audit entries generated by Postgres trigger `trg_cbe_history` |

## Local Development Setup

1. Clone or navigate to the project directory:
   ```bash
   cd f:/AG/Pendency
   ```
2. Copy environment variable file:
   ```bash
   cp .env.local.example .env.local
   ```
3. Set your Supabase credentials in `.env.local`:
   ```env
   NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
   ```
4. Install dependencies:
   ```bash
   npm install
   ```
5. Run dev server:
   ```bash
   npm run dev
   ```
6. Open `http://localhost:3000` in your browser.

## How to Add New Departments or Pendency Types

### Via the In-App Admin Panel (No code changes needed!)
1. Open the app and navigate to **Admin Settings** in the left sidebar.
2. Select the **Departments** or **Pendency Types** tab.
3. Type the new department or lookup type name and click **Add**. The new lookup option will immediately be available across all forms, tables, and filters!

### Via SQL Insert
```sql
-- Add a new department
INSERT INTO departments (name, display_order) VALUES ('Quality Control', 7);

-- Add a new pendency type
INSERT INTO pendency_types (name) VALUES ('Statutory Clearance');
```

## Client-Side Notifications & Realtime Architecture

- **Live Realtime Sync**: Subscribes to Supabase Realtime channels on `pendencies`, `cbe_history`, and `pendency_comments` tables. Edits made on one device propagate live to all active tabs within seconds.
- **15-Minute Fallback Polling**: A background interval refetches data every 15 minutes as a resilience fallback in case of connection loss.
- **Browser Reminders**: Evaluated locally upon data sync. Fires browser `Notification` popups for items with CBE dates due today/tomorrow, overdue items (>3 days critical / >7 days non-critical), and stale items awaiting CBE dates.
- **Deduplication**: Uses `localStorage` key `pendency_tracker_notifications_fired` to ensure reminders fire once per item per day.
